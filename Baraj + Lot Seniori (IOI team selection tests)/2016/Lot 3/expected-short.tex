Editorial – Probabilitatea de a avea două șosete de aceeași culoare
1. Ce trebuie să calculăm ?
Alex extrage K șosete dintr-un sertar cu N șosete.
Fiecare șosetă are

o culoare color_i ∈ [1…N]
o dimensiune – mică (sizeSmall) sau mare (sizeBig).
În momentul în care în sertar mai rămân p șosete cu dimensiunile s₁,…,s_p (fiecare s_i este sizeSmall sau sizeBig), șoseta i este aleasă cu probabilitatea

[ \frac{s_i}{s_1+s_2+\dots+s_p}. ]

După ce Alex a luat exact K șosete, dorim probabilitatea ca cel puțin două dintre ele să aibă aceeași culoare (dimensiunea nu contează).

Problema se poate despărţi în două părţi independente:

Compoziţia dimensiunilor – câte din cele K șosete sunt mici și câte sunt mari.
Coliziunea de culori – dată o combinație (a,b) de a mici şi b mari, care este probabilitatea ca toate culorile să fie diferite?
Răspunsul final este suma, pentru toate perechile (a,b) cu a+b=K, a produsului dintre probabilitatea de a obține exact acea compoziție și probabilitatea ca în acea compoziție să apară o coliziune de culori.

[ \boxed{\displaystyle \text{Ans}= \sum_{a+b=K}; q[a][b];\bigl(1-p[a][b]\bigr) } ]

Unde

q[a][b] – probabilitatea ca în cele K alegeri să fie exact a șosete mici și b mari.
p[a][b] – probabilitatea ca, în orice submulţime de a mici şi b mari, toate culorile să fie distincte.
2. Probabilitatea de a obține o anumită compoziție (a,b)
Numărul total de șosete mici este A, iar al celor mari este B.
La fiecare pas, dacă încă nu am luat i mici și j mari, în sertar mai rămân A‑i mici și B‑j mari, cu greutăţi totale

[ W_{small}= (A-i)\cdot sizeSmall,\qquad W_{big}= (B-j)\cdot sizeBig . ]

Probabilitatea de a alege o șosetă mică în pasul următor este

[ \frac{W_{small}}{W_{small}+W_{big}}, ]

iar cea de a alege o mare este complementul.

Acest proces se modelează perfect printr-un DP bidimensional:

q[0][0] = 1
for i = 0…A
  for j = 0…B
        // alegem încă o mică
        q[i+1][j] += q[i][j] * ( (A-i)*sizeSmall ) /
                               ( (A-i)*sizeSmall + (B-j)*sizeBig )
        // alegem încă o mare
        q[i][j+1] += q[i][j] * ( (B-j)*sizeBig )  /
                               ( (A-i)*sizeSmall + (B-j)*sizeBig )
După K paşi, q[a][b] (cu a+b=K) conţine exact probabilitatea că în selecţie există a mici şi b mari.
Complexitatea este O(A·B) ≤ O(N²) – cu N<200 este trivă.

3. Probabilitatea ca toate culorile să fie diferite (p[a][b])
Această parte nu depinde de greutăţi, ci doar de numărul de șosete de fiecare culoare.
Pentru fiecare culoare c ştim:

cntA[c] – câte mici există în sertar.
cntB[c] – câte mari există în sertar.
Vrem să numărăm modurile în care putem alege a mici și b mari fără să repetăm nicio culoare.
Observăm că, dacă am ales deja x mici și y mari din primele t‑1 culori, atunci pentru a adăuga o șosetă de culoarea t avem două opţiuni:

să nu luăm nicio şosetă de acea culoare,
să luăm exact o mică (alegem una dintre cntA[t] și o combinăm cu poziţia x+1 în secvenţa de mici),
să luăm exact o mare (analog).
Acest lucru duce la un DP peste culori:

DP[0][0][0] = 1
for each colour c = 1…C
   for a = 0…A
      for b = 0…B
          DP[c][a][b] = DP[c-1][a][b]                     // nu luăm culoarea c
          if a>0  DP[c][a][b] += DP[c-1][a-1][b] * cntA[c] * a
          if b>0  DP[c][a][b] += DP[c-1][a][b-1] * cntB[c] * b
Factorii a și b provin din faptul că, odată ce am decis să iau o mică (sau mare) de o anumită culoare, aceasta poate fi poziţionată în oricare dintre locurile deja ocupate de mici (respectiv mari) – alegem poziţia „pentru a nu crea coliziune”.

După ce am procesat toate culorile, DP[C][a][b] este numărul de aranjamente valide (fără culori repetate).

Numărul total de aranjamente (ignoraţi restricţia de culori) este simplu:

[ \text{tot}[a][b] = A\cdot (A-1)\dots (A-a+1); \times; B\cdot (B-1)\dots (B-b+1) ]

adică permutarea a a mici din A și a b mari din B.
Prin urmare

[ p[a][b] = \frac{DP[C][a][b]}{\text{tot}[a][b]} . ]

Acest DP are complexitate O(C·A·B) ≤ O(N³); cu N<200 se încadrează confortabil în limită (≈ 8·10⁶ operaţii).

4. Combinarea celor două rezultate
Pentru fiecare pereche (a,b) cu a+b = K calculăm:

contrib = q[a][b] * (1 - p[a][b])
Ans    += contrib
1-p[a][b] este exact probabilitatea ca cel puțin două șosete să aibă aceeași culoare în acea compoziție.
Suma tuturor contribuţiilor dă răspunsul cerut.

5. Complexitate și precizie
Timp – O(N² + N³) ≈ O(N³). Pentru N ≤ 199 este sub 10⁷ operaţii, uşor de respectat în 1‑2 secunde.
Memorie – 3‑dimensional DP de dimensiuni C·(A+1)·(B+1) (max ~ 200³ ≈ 8 MiB) plus matrici auxiliare. În total < 50 MiB.
Precizie – toate calculele se fac în long double (sau double cu atenție) pentru a respecta eroarea absolută ≤ 10⁻⁵ cerută.
6. Ideea cheie
Separarea dimensiunii de culoare – greutatea de alegere depinde numai de dimensiune, deci distribuţia de mici/mari (q[a][b]) poate fi calculată independent de culori.
Numărarea aranjamentelor fără coliziune – printr-un DP peste culori, unde fiecare culoare poate contribui cu 0, 1 mică sau 1 mare, iar factorii de poziție (a și b) asigură că nu se formează două șosete de aceeași culoare.
Combinaţia prin legea totalelor – probabilitatea finală este suma produselor dintre probabilitatea compoziţiei și probabilitatea de coliziune în acea compoziție.
Aceste observații reduc problema aparent complicată la două DP‑uri bine înţelese și la o simplă sumă, permiţând rezolvarea în timp și memorie adecvate.
